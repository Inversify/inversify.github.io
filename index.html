InversifyJS

A lightweight (4KB) IoC container written in TypeScript.

Star Fork Download

About

Inversify is a lightweight pico inversion of control (IoC) container for TypeScript and JavaScript apps. A pico IoC container uses a class constructor to identify and inject its dependencies.

For this to work, the class needs to declare a constructor that includes everything it needs injected.

The Basics (with TypeScript)

  • 1. Declare interfaces & implementations
  • 2. Bind interfaces to implementations
  • 3. Resolve & inject dependencies

We are going to write SOLID code. We should "depend upon Abstractions. Do not depend upon concretions." so we will start by declaring some interfaces and continue by implementing them. We will start by two classes without dependencies (Foo & Bar). 

interface FooInterface {
  log() : void;
}

interface BarInterface {
  log() : void;
}

interface FooBarInterface {
  log() : void;
}

class Foo implements FooInterface {
  public log(){
    console.log("foo");
  }
}

class Bar implements BarInterface {
  public log(){
    console.log("bar");
  }
}

The class FooBar has two dependencies (FooInterface & BarInterface). Note that the names of arguments in constructor are significant, because the injector uses these to look up the dependencies. 

class FooBar implements FooBarInterface {
  public foo : FooInterface;
  public bar : BarInterface;
  public log(){
    console.log("foobar");
  }
  constructor(FooInterface : FooInterface, BarInterface : BarInterface) {
    this.foo = FooInterface;
    this.bar = BarInterface;
  }
}

Before we can start resolving dependencies we need to create an instance of the inversify Kernel. The Kernel will automatically detect dependencies and will resolve and inject them into a class via its constructor. 

// kernel
var kernel = new inversify.Kernel();

In order to resolve a depencency, the kernel needs to be told which implementation type (classes) to associate with each service type (interfaces). We will use type bindigns for this purpose. A type binding (or just a binding) is a mapping between a service type (an interface), and an implementation type to be used to satisfy such a service requirement. 

// bind
kernel.bind(new inversify.TypeBinding<FooInterface>("FooInterface", Foo));
kernel.bind(new inversify.TypeBinding<BarInterface>("BarInterface", Bar));
kernel.bind(new inversify.TypeBinding<FooBarInterface>("FooBarInterface", FooBar));

var foobar = kernel.resolve<FooBarInterface>("FooBarInterface");

// Foo and bar instances has been injected to foobar via its constructor
foobar.foo.log(); // foo
foobar.bar.log(); // bar
foobar.log();     // foobar
